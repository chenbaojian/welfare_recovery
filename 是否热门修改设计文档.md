# 是否热门修改设计文档

## 一、变更说明

### 1.1 问题描述

当前设计将 `isHot`（是否热门）字段放在面值明细表（`card_product_face_value`）中，意味着每个面值都可以独立设置是否热门。

但实际业务场景中：
- **卡产品维度**："中国移动充值卡"作为一个整体品牌，是否热门应该是该卡产品的属性
- **面值维度**：不同面值可能有不同的可售状态（isSaleable），但热门与否应该是卡产品级别的

### 1.2 变更方案

| 字段 | 原设计（面值维度） | 变更后（设计维度） |
|------|-------------------|-------------------|
| isHot | CardProductFaceValue.isHot | CardProduct.isHot |
| isSaleable | CardProductFaceValue.isSaleable | CardProductFaceValue.isSaleable（保留） |
| discountRate | CardProductFaceValue.discountRate | CardProductFaceValue.discountRate（保留） |
| buyDiscountRate | CardProductFaceValue.buyDiscountRate | CardProductFaceValue.buyDiscountRate（保留） |

## 二、数据库变更

### 2.1 面值明细表变更

从 `card_product_face_value` 表移除 `is_hot` 字段：

```sql
ALTER TABLE `card_product_face_value` DROP COLUMN `is_hot`;
```

### 2.2 卡产品表变更

在 `card_product` 表添加 `is_hot` 字段：

```sql
ALTER TABLE `card_product` ADD COLUMN `is_hot` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否热门: 1-是, 0-否' AFTER `status`;
```

### 2.3 数据迁移

将 `card_product_face_value.is_hot` 数据汇总到 `card_product.is_hot`：

```sql
-- 如果该卡产品有任何面值的is_hot=1，则该卡产品is_hot=1
UPDATE card_product cp
INNER JOIN (
  SELECT card_product_id, MAX(is_hot) as max_is_hot
  FROM card_product_face_value
  GROUP BY card_product_id
) fv ON cp.id = fv.card_product_id
SET cp.is_hot = fv.max_is_hot;
```

## 三、数据层级关系（变更后）

```
CardType (卡类型，如"话费充值卡")
  └── CardProduct (卡产品，如"中国移动充值卡")
        ├── name, sort, status          -- 卡产品级公共属性
        ├── isHot: 1                   -- 是否热门（卡产品级别）
        └── CardProductFaceValue[] (面值明细)
              ├── faceValue: 20         -- 面值金额
              ├── discountRate: 0.95    -- 20元面值专属收卡折扣
              ├── buyDiscountRate: 0.97 -- 20元面值专属卖卡折扣
              ├── isSaleable: 1         -- 20元面值是否可售（面值级别）
              └── sort: 0
              ├── faceValue: 100
              ├── discountRate: 0.93
              ├── buyDiscountRate: 0.95
              ├── isSaleable: 1
              └── sort: 1
              ├── faceValue: 500
              ├── discountRate: 0.90
              ├── buyDiscountRate: 0.93
              ├── isSaleable: 0         -- 500元面值暂停销售
              └── sort: 2
```

## 四、后端接口设计

### 4.1 卡产品相关接口变更

#### 4.1.1 卡产品列表

```
GET /api/admin/card-products/all
```

**响应变更：**

移除 `faceValueCount` 字段，新增 `isHot` 字段：

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "cardTypeId": 1,
        "cardTypeName": "话费充值卡",
        "name": "中国移动",
        "faceValueCount": 5,
        "isHot": 1,
        "sort": 1,
        "status": "ACTIVE"
      }
    ]
  }
}
```

**筛选参数变更：**

| 参数 | 说明 |
|------|------|
| hasHotFaceValue | 移除（不再支持按面值热门筛选） |
| hasSaleableFaceValue | 保留（按面值可售筛选） |
| isHot | 新增（按卡产品热门筛选：1=热门，0=非热门） |

#### 4.1.2 创建卡产品

```
POST /api/admin/card-product/create
```

**请求参数新增：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| isHot | Integer | 否 | 是否热门（0/1），默认0 |

#### 4.1.3 更新卡产品

```
POST /api/admin/card-product/:id/update
```

**请求参数新增：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| isHot | Integer | 否 | 是否热门（0/1） |

#### 4.1.4 切换卡产品热门状态

```
POST /api/admin/card-product/:id/toggle-hot
```

**功能：** 切换卡产品的热门状态（isHot 0↔1）

## 七、影响分析

### 7.1 兼容性考虑
- 旧版本数据迁移：需要将面值级别的 isHot 汇总到卡产品级别
- API 变更：面值相关接口移除 isHot 字段，需要同步更新小程序

### 7.2 业务影响
- 热门推荐：用户在小程序热门推荐中看到的是卡产品（品牌）级别的热门
- 可售状态：不同面值仍然可以独立设置是否可售
