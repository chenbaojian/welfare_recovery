# 回收资产需求文档

## 1. 功能概述

在管理后台侧边栏新增「回收资产」子目录，用于统计和展示所有**已完成状态（SUCCESS）**订单的卡券回收资产数据。该模块聚焦于已成功回收的卡券资产盘点，帮助管理员掌握各类卡券的回收总量、面值金额、卡号卡密等核心信息。

---

## 2. 用户角色

| 角色 | 权限 |
|------|------|
| 超级管理员（SUPER_ADMIN） | 可查看、导出回收资产数据 |
| 管理员（ADMIN） | 可查看、导出回收资产数据 |
| 运营人员（OPERATOR） | 仅可查看回收资产数据 |

## 4. 功能详细说明

### 4.1 统计概览区

页面顶部展示 4 个统计卡片，数据来源为所有状态为 `SUCCESS` 的订单：

| 卡片 | 数据字段 | 说明 |
|------|----------|------|
| 回收总张数 | `totalCount` | 已完成订单的总数量（即回收卡券总张数） |
| 回收总面值 | `totalFaceValue` | 所有已完成订单的 `faceValue` 之和 |
| 回收总金额 | `totalRecycleAmount` | 所有已完成订单的 `recycleAmount` 之和 |
| 卡券种类数 | `cardTypeCount` | 涉及的不同 `cardTypeName` 种类数量 |

**示例展示：**

```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  📦 回收总张数 │ │  💵 回收总面值 │ │  💰 回收总金额 │ │  🎫 卡券种类数 │
│     156      │ │  ¥15,600.00 │ │  ¥12,480.00 │ │      8      │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

### 4.2 卡券分类明细区

按卡券类型（`cardTypeName`）分组展示，每种卡券类型为一个独立的数据区块。

#### 4.2.1 分类汇总行

每种卡券类型展示一行汇总统计：

| 字段 | 说明 |
|------|------|
| 卡券类型名称 | `cardTypeName`（如：话费充值卡、加油卡等） |
| 总张数 | 该类型下已完成订单的数量 |
| 总面值金额 | 该类型下所有订单 `faceValue` 的合计 |
| 总回收金额 | 该类型下所有订单 `recycleAmount` 的合计 |
| 平均折扣率 | `总回收金额 / 总面值金额 × 100%`，反映该类卡券的平均回收折扣 |

**示例：**

| 卡券类型 | 总张数 | 总面值金额 | 总回收金额 | 平均折扣率 |
|----------|--------|-----------|-----------|-----------|
| 话费充值卡 | 45 | ¥4,500.00 | ¥3,600.00 | 80% |
| 加油卡 | 32 | ¥6,400.00 | ¥5,120.00 | 80% |
| 游戏点卡 | 28 | ¥2,800.00 | ¥1,960.00 | 70% |

#### 4.2.2 卡券明细表格

点击某一行分类汇总行的「查看明细」按钮，展开/弹出该类型下所有已完成订单的明细表格，展示每张卡券的核心信息：

| 列名 | 数据来源 | 说明 |
|------|----------|------|
| 订单号 | `orderNo` | 唯一订单编号 |
| 用户昵称 | 关联 User.nickname | 提交该订单的用户昵称 |
| 用户手机号 | 关联 User.phone | 提交该订单的用户手机号 |
| 卡券面值 | `faceValue` | 卡券原始面值金额 |
| 回收金额 | `recycleAmount` | 实际回收入账金额 |
| 卡号 | `cardNo`（解密后） | 卡券卡号，需调用 `decrypt()` 解密后展示 |
| 卡密 | `cardPwd`（解密后） | 卡券卡密，需调用 `decrypt()` 解密后展示 |
| 提交时间 | `createTime` | 用户提交订单的时间 |
| 完结时间 | `completeTime` | 管理员确认完结的时间 |

**安全要求：** 卡号和卡密属于敏感信息，解密后仅在管理后台展示，不得通过普通用户接口返回。页面需标注「敏感信息，仅管理员可见」提示。

#### 4.2.3 面值分布统计

在每种卡券类型的汇总行下方，展示该类型卡券按面值维度的分布统计：

| 面值 | 张数 | 总面值 | 总回收金额 |
|------|------|--------|-----------|
| 50元 | 15 | ¥750.00 | ¥600.00 |
| 100元 | 20 | ¥2,000.00 | ¥1,600.00 |
| 200元 | 10 | ¥2,000.00 | ¥1,600.00 |

## 5. 筛选与查询

### 5.1 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 卡券类型 | 下拉选择 | 篩选特定卡券类型，选项从 `card_type` 表动态获取 |
| 面值范围 | 数值输入 | 最小面值 ~ 最大面值，筛选指定面值区间的卡券 |
| 时间范围 | 日期范围选择器 | 按订单完结时间（`completeTime`）筛选，支持选择起止日期 |
| 用户手机号 | 文本输入 | 搜索特定用户的回收资产 |

### 5.2 筛选逻辑

- 所有筛选条件为 AND 组合关系
- 时间范围筛选基于 `completeTime`（完结时间），而非 `createTime`（提交时间），因为只有完结的订单才构成回收资产
- 卡券类型筛选基于 `cardTypeName`
- 面值范围筛选基于 `faceValue`
- 篩选后统计概览区的数据同步更新

### 5.3 操作按钮

| 按钮 | 功能 |
|------|------|
| 刷新 | 重新加载当前筛选条件下的数据 |
| 导出Excel | 将当前筛选条件下的卡券明细数据导出为 Excel 文件 |

## 7. 数据查询逻辑

### 7.1 核心查询条件

所有回收资产数据查询的基础 WHERE 条件：

```sql
WHERE status = 'SUCCESS'
```

仅查询已完成状态的订单，其他状态（PENDING/PROCESSING/FAILED/CANCELLED）的订单不纳入回收资产统计。

### 7.2 分类汇总查询

按 `cardTypeName` 分组聚合：

```sql
SELECT 
  card_type_name,
  COUNT(*) AS total_count,
  SUM(face_value) AS total_face_value,
  SUM(recycle_amount) AS total_recycle_amount,
  ROUND(SUM(recycle_amount) / SUM(face_value) * 100, 2) AS avg_discount_rate
FROM order
WHERE status = 'SUCCESS'
  [AND 篮选条件]
GROUP BY card_type_name
ORDER BY total_count DESC
```

### 7.3 面值分布查询

在每种卡券类型内，按 `faceValue` 进一步分组：

```sql
SELECT 
  face_value,
  COUNT(*) AS count,
  SUM(face_value) AS total_face_value,
  SUM(recycle_amount) AS total_recycle_amount
FROM order
WHERE status = 'SUCCESS' AND card_type_name = ?
  [AND 篮选条件]
GROUP BY face_value
ORDER BY face_value ASC
```

### 7.4 明细列表查询

```sql
SELECT * FROM order
WHERE status = 'SUCCESS'
  [AND 篮选条件]
ORDER BY complete_time DESC
LIMIT ? OFFSET ?
```

查询后需：
1. 对 `cardNo` 和 `cardPwd` 调用 `decrypt()` 解密
2. 关联 `User` 表获取 `nickname` 和 `phone`

## 9. 后端实现要点

### 9.1 路由注册

// 回收资产

在 `server/src/services/admin.js` 中新增：

| 方法 | 功能 | 核心逻辑 |

## 11. 安全与权限

| 安全要求 | 说明 |
|----------|------|
| 管理员认证 | 所有回收资产接口必须经过 `adminAuth` 中间件验证 |
| 卡号卡密加密存储 | 数据库中 `cardNo`/`cardPwd` 以 AES-256-CBC 加密存储，管理后台展示时解密 |
| 敏感信息提示 | 前端页面在展示卡号卡密区域标注「⚠️ 敏感信息，仅管理员可见」 |
| 导出权限控制 | 导出Excel功能仅 SUPER_ADMIN 和 ADMIN 角色可用，OPERATOR 角色无导出权限 |
| 导出文件安全 | 导出的Excel文件中包含卡号卡密明文，需提醒管理员妥善保管 |

| 场景 | 优化方案 |
|------|----------|
| 大量订单数据聚合 | 使用数据库 `GROUP BY` + `SUM` 聚合查询，避免全量加载到内存计算 |
| 明细列表分页 | 使用 `LIMIT` + `OFFSET` 分页查询，默认每页20条 |
| 用户信息关联 | 批量查询用户信息（收集所有userId后一次性查询），避免N+1查询问题 |
| 导出大数据量 | 当数据量超过1000条时，采用流式写入Excel，避免内存溢出 |
| 卡券类型筛选 | 前端下拉框数据从 `card_type` 表动态获取，不硬编码 |

## 13. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 无已完成订单 | 统计卡片显示0，分类汇总表格显示「暂无回收资产数据」 |
| 卡号卡密解密失败 | 明细表格中卡号/卡密列显示「解密失败」，不影响其他数据展示 |
| 导出数据为空 | 提示「当前筛选条件下无数据可导出」，不生成空Excel文件 |
| 篩选条件无效 | 忽略无效参数，使用默认值查询 |
| 权限不足 | 返回401，前端跳转登录页 |

## 14. 交互流程

### 14.1 页面加载流程

```
1. 管理员点击侧边栏「回收资产」菜单
2. 前端调用 GET /api/admin/assets/summary（无筛选参数）
3. 后端查询所有SUCCESS订单的汇总统计
4. 前端渲染统计概览卡片 + 分类汇总表格
5. 同时调用 GET /api/admin/card-types 获取卡券类型列表，填充筛选下拉框
```

### 14.2 查看面值分布流程

```
1. 管理员点击分类汇总行中的「面值分布」按钮
2. 前端从summary接口返回的faceValueDistribution数据中获取该类型的面值分布
3. 在分类汇总表格下方展开面值分布表格
```

### 14.3 查看卡券明细流程

```
1. 管理员点击分类汇总行中的「查看明细」按钮
2. 前端打开卡券明细弹窗
3. 调用 GET /api/admin/assets/detail?cardTypeName=xxx&page=1&pageSize=20
4. 后端查询该类型下SUCCESS订单明细，解密卡号卡密，关联用户信息
5. 前端渲染明细表格（含卡号卡密明文）
6. 支持分页浏览
```

### 14.4 篩选查询流程

```
1. 管理员设置筛选条件（卡券类型/面值范围/时间范围/手机号）
2. 点击「刷新」按钮
3. 前端将筛选参数附加到 GET /api/admin/assets/summary 请求
4. 后端根据筛选条件查询SUCCESS订单汇总统计
5. 前端更新统计概览卡片 + 分类汇总表格
```

### 14.5 导出Excel流程

```
1. 管理员点击「导出Excel」按钮
2. 前端将当前筛选参数附加到 GET /api/admin/assets/export 请求
3. 后端查询符合条件的SUCCESS订单明细
4. 生成Excel文件（含卡号卡密解密后的明文数据）
5. 返回文件流，前端触发浏览器下载
6. 文件名格式：回收资产_YYYYMMDD.xlsx
```

## 15. 数据模型关系

```
Order (status = 'SUCCESS')
  ├── cardTypeName  → 卡券类型名称（分组维度）
  ├── faceValue     → 卡券原始面值（金额维度）
  ├── recycleAmount → 实际回收金额（资产价值）
  ├── cardNo        → 卡号（加密存储，展示时解密）
  ├── cardPwd       → 卡密（加密存储，展示时解密）
  ├── completeTime  → 完结时间（时间维度）
  ├── userId        → 关联 User
  │     ├── nickname  → 用户昵称
  │     └── phone     → 用户手机号
  └── orderNo       → 订单号（唯一标识）

CardType
  ├── id            → 卡券类型ID
  ├── name          → 卡券类型名称（与Order.cardTypeName对应）
  ├── category      → 卡券分类
  └── discountRate  → 折扣率（参考值）
```
