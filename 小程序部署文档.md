## 1. 环境要求

### 1.1 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 40 GB SSD | 80 GB SSD |
| 带宽 | 5 Mbps | 10 Mbps |
| 操作系统 | CentOS 7.9+ / Ubuntu 20.04+ | Ubuntu 22.04 LTS |

### 1.2 软件依赖

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | >= 16.0.0 | 后端运行环境 |
| MySQL | >= 5.7 | 数据库 |
| Redis | >= 6.0 | 缓存（可选，当前使用内存缓存） |
| Nginx | >= 1.20 | 反向代理（推荐） |
| PM2 | >= 5.0 | Node.js 进程管理（推荐） |

### 1.3 域名要求

- 一个已备案的域名（如 `api.yourdomain.com`）
- 微信小程序要求所有请求域名必须为 HTTPS 且已备案

## 3. 服务器环境准备

### 3.1 连接服务器

```bash
ssh root@your_server_ip
```

### 3.2 安装 Node.js

```bash
# 使用 NodeSource 安装 Node.js 18 LTS
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v   # 应输出 v18.x.x
npm -v    # 应输出 9.x.x

# 安装 PM2 进程管理器
npm install -g pm2
```

### 3.3 安装 MySQL

```bash
# Ubuntu 22.04 安装 MySQL 8.0
sudo apt-get update
sudo apt-get install -y mysql-server

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置（设置 root 密码等）
sudo mysql_secure_installation
```

### 3.4 创建数据库和用户

```bash
# 登录 MySQL
sudo mysql -u root -p

# 创建数据库
CREATE DATABASE welfare_recovery CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建专用用户（替换 your_password 为强密码）
CREATE USER 'welfare'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON welfare_recovery.* TO 'welfare'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3.5 安装 Redis（可选）

```bash
sudo apt-get install -y redis-server
sudo systemctl start redis
sudo systemctl enable redis
```

### 3.6 安装 Nginx

```bash
sudo apt-get install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

## 4. 后端服务部署

### 4.1 上传项目文件到服务器

```bash
# 在服务器上创建项目目录
mkdir -p /opt/welfare-recovery
cd /opt/welfare-recovery

# 方式一：使用 scp 上传（在本地开发机执行）
scp -r D:\福利收回\server root@your_server_ip:/opt/welfare-recovery/
scp -r D:\福利收回\server\admin root@your_server_ip:/opt/welfare-recovery/

# 方式二：使用 git 部署（如果使用代码仓库）
# git clone https://your-repo-url.git .
```

### 4.2 安装依赖

```bash
cd /opt/welfare-recovery/server
npm install --production
```

### 4.3 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
nano .env
```

**.env 文件配置说明：**

```ini
# 环境配置（production 表示生产环境）
NODE_ENV=production
PORT=3000

# 数据库配置（使用刚才创建的数据库和用户）
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=welfare_recovery
DB_USER=welfare
DB_PASSWORD=你的数据库密码

# Redis配置（如果不使用 Redis 可留空）
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT配置（生成一个随机字符串作为密钥）
JWT_SECRET=生成一个随机字符串，建议使用 openssl rand -hex 32 生成
JWT_EXPIRE=7d

# 微信小程序配置（在微信公众平台获取）
WX_APPID=你的小程序AppID
WX_SECRET=你的小程序AppSecret

# 加密配置（用于卡号卡密加密，生成一个32位随机字符串）
CRYPTO_KEY=生成一个32位随机字符串

# 日志配置
LOG_LEVEL=info
LOG_FILE=logs/app.log

# CORS配置（允许的前端域名）
CORS_ORIGIN=*
```

生成密钥的命令：

```bash
# 生成 JWT_SECRET
openssl rand -hex 32

# 生成 CRYPTO_KEY（需要32位）
openssl rand -hex 16
```

### 4.4 启动后端服务

```bash
# 使用 PM2 启动
cd /opt/welfare-recovery/server
pm2 start src/app.js --name welfare-server

# 设置 PM2 开机自启
pm2 startup
pm2 save

# 查看运行状态
pm2 status
pm2 logs welfare-server
```

### 4.5 验证服务运行

```bash
# 本地测试
curl http://localhost:3000/health

# 预期返回
# {"status":"ok","timestamp":"2026-05-10T12:00:00.000Z"}
```

## 5. 微信小程序配置与上传

### 5.1 修改小程序 API 地址

编辑 `miniprogram/config/index.js`，将生产环境的 `baseUrl` 改为你的服务器域名：

```javascript
production: {
  baseUrl: 'https://api.yourdomain.com/api',  // 改为你的域名
  timeout: 10000
}
```

### 5.2 配置微信公众平台

登录 [微信公众平台](https://mp.weixin.qq.com/)，进入小程序管理：

#### 5.2.1 开发管理 → 开发设置

- **AppID**：记录下 AppID，用于 `.env` 配置
- **AppSecret**：生成并记录，用于 `.env` 配置
- **服务器域名** → **request合法域名**：
  ```
  https://api.yourdomain.com
  ```
- **uploadFile合法域名**（如果使用文件上传）：
  ```
  https://api.yourdomain.com
  ```

#### 5.2.2 小程序类目设置

确保小程序已设置合适的服务类目（如"闲置交易"或"卡券回收"），否则部分功能可能审核不通过。

### 5.3 上传小程序代码

**方式一：微信开发者工具上传（推荐）**

1. 打开微信开发者工具
2. 打开项目（选择 `D:\福利收回\miniprogram` 目录）
3. 点击工具栏的 **上传** 按钮
4. 填写版本号和更新说明
5. 上传成功后，在微信公众平台 → 版本管理 中提交审核

**方式二：命令行上传（CI/CD）**

```bash
# 安装 miniprogram-ci
npm install -g miniprogram-ci

# 上传代码（需要私钥文件，在微信公众平台下载）
miniprogram-ci upload \
  --pp ./miniprogram \
  --pkp ./private.key \
  --appid wx8e0a92186bf86b07 \
  --uv 1.0.0 \
  --threads 1
```

### 5.4 提交审核与发布

1. 在微信公众平台 → **版本管理** 中找到上传的版本
2. 点击 **提交审核**
3. 填写审核说明（描述功能变更）
4. 审核通过后，点击 **发布**

## 7. Nginx 反向代理配置（推荐）

### 7.1 创建 Nginx 配置文件

```bash
sudo nano /etc/nginx/sites-available/welfare-recovery
```

### 7.2 配置文件内容

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;

    # HTTP 自动跳转 HTTPS（如果已配置证书）
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    # SSL 证书配置（见第8节）
    ssl_certificate /etc/nginx/ssl/api.yourdomain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/api.yourdomain.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 客户端请求大小限制
    client_max_body_size 20m;

    # 日志配置
    access_log /var/log/nginx/welfare-access.log;
    error_log /var/log/nginx/welfare-error.log;

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    # 管理后台静态文件
    location /admin/ {
        proxy_pass http://127.0.0.1:3000/admin/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:3000/health;
        proxy_set_header Host $host;
    }
}
```

### 7.3 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/welfare-recovery /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

## 8. HTTPS 证书配置

### 8.1 使用 Certbot 自动申请 Let's Encrypt 证书（推荐）

```bash
# 安装 Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# 申请证书（会自动修改 Nginx 配置）

# 测试自动续期

```

### 8.2 手动配置证书

如果已有证书文件，将证书上传到服务器：

```bash
# 创建证书目录
sudo mkdir -p /etc/nginx/ssl

# 上传证书文件（在本地执行）
scp /path/to/your/cert.pem root@your_server_ip:/etc/nginx/ssl/api.yourdomain.com.pem
scp /path/to/your/key.pem root@your_server_ip:/etc/nginx/ssl/api.yourdomain.com.key
```

```bash

tail -f /opt/welfare-recovery/server/logs/app.log

# Nginx 访问日志
tail -f /var/log/nginx/welfare-access.log
tail -f /var/log/nginx/welfare-error.log
```

### 10.3 数据库备份

创建定时备份脚本 `/opt/welfare-recovery/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/opt/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="welfare_recovery"
DB_USER="welfare"
DB_PASS="你的数据库密码"

mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/${DB_NAME}_$DATE.sql

# 删除30天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete

echo "数据库备份完成: ${DB_NAME}_$DATE.sql"
```

设置定时任务：

```bash
chmod +x /opt/welfare-recovery/backup.sh

# 添加到 crontab（每天凌晨3点备份）
crontab -e
# 添加以下行：
0 3 * * * /opt/welfare-recovery/backup.sh
```

### 10.4 服务监控

创建健康检查脚本 `/opt/welfare-recovery/healthcheck.sh`：

```bash
#!/bin/bash
URL="https://api.yourdomain.com/health"
RESPONSE=$(curl -s -o /dev/null -w "%{http_code}" $URL)

if [ $RESPONSE -ne 200 ]; then
    echo "服务异常，正在重启..."
    cd /opt/welfare-recovery/server && pm2 restart welfare-server
    echo "服务已重启"
else
    echo "服务正常"
fi
```

### 10.5 服务器安全配置

```bash
# 配置防火墙（仅开放必要端口）
sudo ufw allow 22/tcp      # SSH
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS
sudo ufw allow 3306/tcp    # MySQL（仅内网访问）
sudo ufw enable

# 禁止 root 密码登录 SSH（推荐使用密钥登录）
sudo nano /etc/ssh/sshd_config
# 修改：PermitRootLogin prohibit-password
# 修改：PasswordAuthentication no
sudo systemctl restart sshd
```

## 11. 常见问题排查

### 11.1 小程序无法请求接口

**现象**：小程序提示"网络异常"或"请求失败"

**排查步骤**：

1. 检查微信公众平台 → 开发设置 → 服务器域名 是否已配置
2. 确认域名已备案
3. 确认 HTTPS 证书有效
4. 使用浏览器访问 `https://api.yourdomain.com/health` 测试
5. 查看 Nginx 错误日志：`tail -f /var/log/nginx/welfare-error.log`

### 11.2 数据库连接失败

**现象**：服务启动报错 "数据库连接失败"

**排查步骤**：

```bash
# 检查 MySQL 是否运行
sudo systemctl status mysql

# 测试数据库连接
mysql -u welfare -p -h 127.0.0.1 welfare_recovery

# 检查 .env 配置
cat /opt/welfare-recovery/server/.env | grep DB_
```

### 11.3 微信登录失败

**现象**：用户点击微信登录无反应或提示错误

**排查步骤**：

1. 确认 `.env` 中的 `WX_APPID` 和 `WX_SECRET` 正确
2. 确认微信公众平台已开启"小程序登录"功能
3. 查看后端日志：`pm2 logs welfare-server`
4. 常见错误：`errcode: 40001` 表示 AppSecret 错误

### 11.4 手机号获取失败

**现象**：手机号快捷登录无法获取手机号

**排查步骤**：

1. 确认微信公众平台已开通"获取手机号"接口权限
2. 确认小程序已绑定微信认证服务号（需要认证）
3. 查看后端日志中的 access_token 获取情况

### 11.5 管理后台无法访问

**现象**：访问 `https://api.yourdomain.com/admin` 显示 404

**排查步骤**：

```bash
# 检查 admin 目录是否存在
ls -la /opt/welfare-recovery/server/admin/

# 确认 Nginx 配置正确
sudo nginx -t
```
