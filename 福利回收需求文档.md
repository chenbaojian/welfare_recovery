# 福利回收微信小程序需求设计文档

## 一、项目概述

### 1.1 项目背景
福利回收小程序旨在为用户提供便捷的卡券回收服务，将闲置的各类卡券、积分兑换为现金，实现资源变现。

### 1.2 项目目标
- 提供一站式卡券回收服务
- 简化用户操作流程，提升用户体验
- 保障交易安全，支持实名认证
- 实现快速结算，资金及时到账

### 1.3 支持回收类型

| 类别 | 具体类型 |
|------|----------|
| 通信类 | 电话卡 |
| 出行类 | 加油卡、出行券 |
| 娱乐类 | 游戏卡、电影券、影音券 |
| 电商类 | 电商卡、京东E卡 |
| 生活类 | 商超卡、美食券 |
| 积分类 | 美团企业积分、劳保积分、携程积分 |

---

## 二、用户角色与权限

### 2.1 用户角色
| 角色 | 说明 | 权限 |
|------|------|------|
| 游客 | 未登录用户 | 浏览卡券类型、查看回收价格 |
| 普通用户 | 已登录未实名 | 提交订单（需先实名认证） |
| 认证用户 | 已实名认证 | 完整使用所有功能 |

### 2.2 权限控制流程
```
游客 → 授权登录 → 普通用户 → 实名认证 → 认证用户 → 提交订单
```

### 3.2 核心功能说明

#### 3.2.1 登录模块
**功能描述**：支持两种登录方式
- **微信授权登录**：获取用户微信基本信息（头像、昵称）
- **手机号授权登录**：通过微信授权获取用户手机号

**业务流程**：
1. 用户打开小程序
2. 判断登录状态，未登录则显示登录引导
3. 用户选择登录方式
4. 授权成功后获取用户信息
5. 检查实名状态，未实名则引导认证

#### 3.2.2 实名认证模块
**功能描述**：用户身份验证，保障交易安全

**输入信息**：
- 真实姓名
- 身份证号码

**验证规则**：
- 姓名：2-20个中文字符
- 身份证号：18位有效身份证号码
- 调用第三方接口验证真实性

#### 3.2.3 卡券回收模块
**功能描述**：核心交易功能

**操作流程**：
1. 选择卡券类型
2. 选择面值档位
3. 输入卡号和卡密
4. 确认回收价格
5. 提交订单

### 6.2 交互状态说明

#### 6.2.1 登录交互

| 交互动作 | 系统响应 | 状态变化 |
|----------|----------|----------|
| 点击"微信一键登录" | 调用wx.login获取code | 显示loading |
| 授权成功 | 获取用户信息，跳转首页 | 登录态→已登录 |
| 授权失败 | 提示"授权失败，请重试" | 保持未登录 |
| 点击"手机号登录" | 弹出手机号授权弹窗 | 等待授权 |
| 手机号授权成功 | 获取手机号，自动登录 | 登录态→已登录 |

#### 6.2.2 实名认证交互

| 交互动作 | 系统响应 | 校验规则 |
|----------|----------|----------|
| 输入姓名 | 实时校验格式 | 2-20个中文字符 |
| 输入身份证号 | 实时校验格式 | 18位身份证号 |
| 点击"提交认证" | 显示loading，调用验证接口 | 提交前校验所有字段 |
| 认证成功 | 提示"认证成功"，跳转上一页 | 实名状态→已认证 |
| 认证失败 | 提示具体错误原因 | 保持未认证 |

#### 6.2.3 卡券回收交互

| 交互动作 | 系统响应 | 状态变化 |
|----------|----------|----------|
| 选择卡券类型 | 跳转对应详情页 | 记录卡券类型 |
| 选择面值 | 高亮选中项，计算回收金额 | 更新回收金额 |
| 输入卡号 | 实时校验格式 | 存储卡号 |
| 输入卡密 | 实时校验格式 | 存储卡密 |
| 点击"提交订单" | 弹出确认弹窗 | 等待确认 |
| 确认提交 | 显示loading，创建订单 | 订单状态→处理中 |
| 订单成功 | 提示成功，跳转订单详情 | 订单状态→已完成 |
| 订单失败 | 提示失败原因 | 订单状态→已取消 |

### 6.3 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 网络异常 | 提示"网络异常，请检查网络后重试" |
| 登录过期 | 自动跳转登录页，提示"登录已过期" |
| 卡号卡密错误 | 提示"卡号或卡密错误，请核对后重试" |
| 卡券已使用 | 提示"该卡券已使用，无法回收" |
| 面值不匹配 | 提示"卡券面值与选择不符，请重新选择" |
| 实名认证失败 | 提示具体原因，支持重新提交 |

## 九、安全设计

### 9.1 数据安全
- 敏感信息（身份证号、卡号、卡密）采用AES加密存储
- 传输层采用HTTPS加密
- 用户token设置有效期，定期刷新

### 9.2 接口安全
- 所有接口需验证token有效性
- 关键接口（提交订单、提现）需验证实名状态
- 接口限流，防止恶意请求

### 9.3 业务安全
- 实名认证调用第三方接口验证真实性
- 卡券验证调用对应平台接口
- 订单金额计算在服务端完成，防止篡改
