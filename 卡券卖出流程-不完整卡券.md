# 卡券卖出流程 - 不完整卡券检查报告

## 检查时间：2026-05-10

## 一、检查范围

卖家模式下所有12大类卡券的完整卖出流程：首页入口 → 列表页(如有子类型) → 详情页/提交页 → 创建订单 → 后端接口

## 二、整体结论

所有卡券的**核心卖出链路**（首页入口→列表→详情→提交订单→后端接口）是完整的，但存在以下统一问题：

---

## 三、问题清单

### 问题1：9个详情页缺少前端登录状态检查（高优先级）

**现状**：仅 phone、gas、laobao 三个详情页在 `onSubmitOrder()` 中有 `isLoggedIn` 检查，其余9个页面直接进入表单验证，未登录用户可看到提交按钮并点击，直到后端返回401才报错，用户体验差。

**缺少登录检查的页面**：

| 序号 | 卡券类型 | 页面路径 | cardTypeId |
|------|---------|---------|------------|
| 1 | 游戏卡 | pages/game/detail/detail.js | 3 |
| 2 | 影音券 | pages/music/detail/detail.js | 12 |
| 3 | 电商卡 | pages/ecommerce/detail/detail.js | 13 |
| 4 | 京东E卡 | pages/jd/detail/detail.js | 4 |
| 5 | 商超卡 | pages/supermarket/detail/detail.js | 5 |
| 6 | 美食券 | pages/food/detail/detail.js | 7 |
| 7 | 美团企业积分 | pages/meituan/detail/detail.js | 9 |
| 8 | 携程积分 | pages/ctrip/detail/detail.js | 11 |
| 9 | 融晟携程卡包 | pages/rsctrip/detail/detail.js | 8 |

**已有登录检查的页面**（仅3个）：
- phone/detail/detail.js ✅
- gas/detail/detail.js ✅
- laobao/detail/detail.js ✅

**修复方案**：在9个页面的 `onSubmitOrder()` 方法开头添加与 phone/gas 一致的登录检查逻辑。

---

### 问题2：所有详情页缺少前端实名认证检查（中优先级）

**现状**：首页 `onSelectCard()` 已有登录+实名认证双重检查，但所有详情页的 `onSubmitOrder()` 中均无实名认证检查。如果用户通过URL直接访问详情页，可绕过首页的认证拦截。

**后端保护**：`POST /api/order/create` 路由已有 `auth + verified` 中间件保护，但开发环境 `verified` 中间件会跳过检查（`NODE_ENV=development` 时直接 next），所以开发环境下后端也不拦截。

**修复方案**：在所有12个详情页的 `onSubmitOrder()` 中添加实名认证检查（与首页 `onSelectCard` 一致的逻辑）。

---

### 问题3：rsctrip 详情页 cardNo 传固定字符串（低优先级）

**现状**：`pages/rsctrip/detail/detail.js` 提交订单时 `cardNo: 'rsctrip'`，因为融晟携程卡包只有卡密没有卡号。后端 Order 模型 `cardNo` 字段 `allowNull: false`，所以前端用占位字符串。

**影响**：数据库中存储的卡号无实际意义，管理后台展示时可能造成混淆。

**修复方案**：后端 Order 模型 `cardNo` 改为 `allowNull: true`，前端 rsctrip 不传 cardNo 或传空字符串。

---

### 问题4：通用详情页缺少登录检查（中优先级）

**现状**：`pages/detail/detail.js` 的 `onSubmit()` 方法无登录检查和实名认证检查。虽然该页面目前作为兜底页面不会被首页触发，但通过URL直接访问仍可使用。

**修复方案**：添加与专用详情页一致的登录+实名认证检查。

---

## 四、已有登录检查的参考代码（phone/detail）

```javascript
onSubmitOrder() {
  if (!this.data.isLoggedIn) {
    wx.showModal({
      title: '提示',
      content: '请先登录后再进行操作',
      confirmText: '去登录',
      success: (res) => {
        if (res.confirm) {
          wx.navigateTo({ url: '/pages/login/login' });
        }
      }
    });
    return;
  }
  // ... 后续逻辑
}
```

## 五、首页实名认证检查参考代码

```javascript
const userInfo = this.data.userInfo;
if (userInfo && !userInfo.isVerified) {
  wx.showModal({
    title: '提示',
    content: '请先完成实名认证后再进行操作',
    confirmText: '去认证',
    success: (res) => {
      if (res.confirm) {
        wx.navigateTo({ url: '/pages/verify/verify' });
      }
    }
  });
  return;
}
```
