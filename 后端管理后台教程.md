# 福利回收 - 后端管理后台使用教程

## 一、系统概述

福利回收管理后台是运营人员使用的后台管理系统，用于审核用户提交的卡券回收订单。核心业务流程为：

> **用户提交卡券 → 管理员审核核验 → 完结入账 / 拒绝退回**

管理员在后台查看用户订单，核验卡券信息无误后，手动点击「确认完结」，系统自动将回收金额入账到用户余额并记录流水；若卡券有问题，则点击「拒绝」并填写原因。

## 二、功能介绍

### 2.1 管理员登录

- 支持用户名 + 密码登录
- 默认管理员账号：`admin`，密码：`admin123`
- 登录后生成 JWT Token，后续请求自动携带
- 支持三种角色：`SUPER_ADMIN`（超级管理员）、`ADMIN`（管理员）、`OPERATOR`（运营）

### 2.2 数据概览（首页）

进入后台首页，展示以下统计数据：

| 统计项 | 说明 |
|--------|------|
| 今日订单 | 当日创建的订单总数 |
| 待处理订单 | 状态为 PENDING 的订单数（需重点关注） |
| 处理中订单 | 状态为 PROCESSING 的订单数 |
| 今日回收金额 | 当日已完结订单的回收金额总和 |
| 总用户数 | 平台注册用户总数 |
| 累计回收金额 | 所有已完结订单的回收金额总和 |

### 2.3 订单管理（核心功能）

#### 订单列表

- 支持按状态筛选：全部 / 待处理 / 处理中 / 已完成 / 已失败 / 已取消
- 支持按订单号搜索
- 分页展示，每页 10 条
- 表格展示：订单号、用户昵称、卡券类型、面值、回收金额、状态、创建时间

#### 订单详情

点击「详情」按钮，弹窗展示完整订单信息：

- **订单信息**：订单号、状态、卡券类型、面值、回收金额、创建时间
- **卡券信息**：卡号（解密后明文）、卡密（解密后明文）——供管理员核验
- **用户信息**：用户昵称、手机号、真实姓名
- **失败原因**：已拒绝订单会显示拒绝原因

#### 确认完结（核心操作）

1. 在订单列表或详情弹窗中，对「待处理」或「处理中」状态的订单点击「完结」按钮
2. 弹出确认提示：「确认完结此订单？完结后回收金额将入账到用户余额。」
3. 确认后系统执行以下操作（**事务保证一致性**）：
   - 更新订单状态为 `SUCCESS`（已完成）
   - 记录完成时间
   - 将回收金额入账到用户余额（`balance += recycleAmount`）
   - 更新用户累计回收金额（`totalRecycle += recycleAmount`）
   - 更新用户订单数量（`orderCount += 1`）
   - 记录余额流水（`balance_log` 表，类型为 `RECYCLE_INCOME`）
4. 小程序端用户下次查看订单时，状态自动变为「已完成」，余额自动更新

#### 拒绝订单

1. 对「待处理」或「处理中」状态的订单点击「拒绝」按钮
2. 弹出拒绝原因输入框，填写原因（如：卡券无效、卡密错误、卡号不存在等）
3. 确认后系统执行：
   - 更新订单状态为 `FAILED`（已失败）
   - 记录失败原因（`failReason`）
   - 记录完成时间
4. 小程序端用户查看订单时，会显示「已失败」及失败原因，并提供「联系客服」按钮

### 2.4 用户管理

#### 用户列表

- 支持按手机号、昵称搜索
- 分页展示
- 表格展示：ID、昵称、手机号、认证状态、余额、累计回收、订单数、注册时间

#### 用户详情

点击「详情」按钮，弹窗展示：

- **基本信息**：ID、昵称、手机号、认证状态、余额、累计回收、订单数、注册时间
- **订单统计**：各状态的订单数量和金额汇总
- **余额流水**：最近 20 条余额变动记录（回收收入 / 提现支出 / 管理员调整）

## 三、订单状态说明

| 状态值 | 中文名 | 颜色 | 说明 |
|--------|--------|------|------|
| `PENDING` | 待处理 | 🟡 黄色 | 用户刚提交，等待管理员审核 |
| `PROCESSING` | 处理中 | 🔵 蓝色 | 管理员正在核验（预留状态） |
| `SUCCESS` | 已完成 | 🟢 绿色 | 管理员确认完结，金额已入账 |
| `FAILED` | 已失败 | 🔴 红色 | 管理员拒绝，附失败原因 |
| `CANCELLED` | 已取消 | ⚪ 灰色 | 用户主动取消（仅待处理状态可取消） |

**状态流转规则：**

```
用户提交 → PENDING（待处理）
              ↓
管理员完结 → SUCCESS（已完成，金额入账）
              ↓
管理员拒绝 → FAILED（已失败，附原因）
              ↓
用户取消   → CANCELLED（已取消，仅PENDING状态可取消）
```

- 用户名：`admin`

## 五、API接口文档

所有管理后台接口路径前缀为 `/api/admin`，需登录后携带 Token。

### 5.1 管理员登录

```
POST /api/admin/login
```

**请求参数：**
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "adminInfo": {
      "id": 1,
      "username": "admin",
      "realName": "系统管理员",
      "role": "SUPER_ADMIN"
    }
  }
}
```

### 5.2 获取管理员信息

```
GET /api/admin/info
Authorization: Bearer <token>
```

### 5.3 数据概览

```
GET /api/admin/dashboard
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 200,
  "data": {
    "todayOrders": 5,
    "pendingOrders": 3,
    "processingOrders": 1,
    "totalUsers": 100,
    "todayUsers": 2,
    "totalRecycle": 1256.50,
    "todayRecycle": 95.00
  }
}
```

### 5.4 订单列表

```
GET /api/admin/order/list?page=1&pageSize=10&status=PENDING&orderNo=20240101
Authorization: Bearer <token>
```

**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认1 |
| pageSize | int | 否 | 每页数量，默认10 |
| status | string | 否 | 状态筛选：PENDING/PROCESSING/SUCCESS/FAILED/CANCELLED |
| orderNo | string | 否 | 订单号搜索 |
| userId | int | 否 | 用户ID筛选 |
| startDate | string | 否 | 开始日期 |
| endDate | string | 否 | 结束日期 |

**响应字段说明：**
- `cardNoDecrypt`：解密后的卡号明文（供管理员核验）
- `cardPwdDecrypt`：解密后的卡密明文（供管理员核验）
- `userPhone`、`userNickname`、`userRealName`：关联用户信息

### 5.5 订单详情

```
GET /api/admin/order/detail?id=1
Authorization: Bearer <token>
```

### 5.6 确认完结订单（核心接口）

```
POST /api/admin/order/complete
Authorization: Bearer <token>
```

**请求参数：**
```json
{
  "orderId": 1
}
```

**响应：**
```json
{
  "code": 200,
  "message": "订单已完结，金额已入账",
  "data": {
    "orderId": 1,
    "orderNo": "202401010001",
    "recycleAmount": 95.00,
    "balanceAfter": 1256.50
  }
}
```

**后端执行逻辑（事务）：**
1. 校验订单状态（仅 PENDING / PROCESSING 可完结）
2. 更新订单状态为 SUCCESS，记录完成时间
3. 获取用户当前余额，计算新余额
4. 更新用户余额、累计回收金额、订单数量
5. 写入余额流水记录（type=RECYCLE_INCOME）
6. 事务提交，任何一步失败则全部回滚

### 5.7 拒绝订单

```
POST /api/admin/order/reject
Authorization: Bearer <token>
```

**请求参数：**
```json
{
  "orderId": 1,
  "failReason": "卡券无效，卡密错误"
}
```

### 5.8 用户列表

```
GET /api/admin/user/list?page=1&pageSize=10&phone=13800138000&nickname=测试
Authorization: Bearer <token>
```

### 5.9 用户详情

```
GET /api/admin/user/detail?id=1
Authorization: Bearer <token>
```

**响应包含：**
- `userInfo`：用户基本信息
- `orderStats`：各状态订单统计
- `balanceLogs`：最近20条余额流水

## 六、如何启动

### 6.1 环境要求

| 依赖 | 版本要求 |
|------|----------|
| Node.js | ≥ 16.0.0 |
| MySQL | ≥ 5.7 |
| npm | ≥ 8.0 |

### 6.2 初始化数据库

1. 启动 MySQL 服务
2. 执行数据库初始化脚本：

```bash
mysql -u root -p < server/database/init.sql
```

该脚本会：
- 创建 `welfare_recovery` 数据库
- 创建所有表（user、card_type、order、bank、withdraw、admin、balance_log）
- 初始化管理员账号（admin/admin123）
- 初始化卡券类型数据

### 6.3 配置环境变量

编辑 `server/.env` 文件，确认以下配置：

```env
# 数据库配置（根据实际修改）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=welfare_recovery
DB_USER=root
DB_PASSWORD=你的MySQL密码

# JWT配置
JWT_SECRET=welfare_recovery_jwt_secret_key_2024
JWT_EXPIRE=7d

# 服务端口
PORT=3000
```

### 6.4 安装后端依赖

```bash
cd server
npm install
```

### 6.5 启动后端服务

**开发模式（自动重启）：**
```bash
cd server
npm run dev
```

**生产模式：**
```bash
cd server
npm start
```

启动成功后控制台输出：
```
服务启动成功，端口: 3000
环境: development
数据库连接成功
```

### 6.6 访问管理后台

后端服务启动后，管理后台前端页面已内置在服务中，无需单独启动。

**访问地址：**

```
http://localhost:3000/admin
```

打开浏览器访问上述地址，即可进入管理后台登录页面。

### 6.7 启动小程序

1. 打开微信开发者工具
2. 导入项目目录：`D:\福利收回\miniprogram`
3. AppID 使用：`wx8e0a92186bf86b07`（或测试 AppID）
4. 编译运行

## 七、使用流程（图文说明）

### 步骤1：管理员登录

1. 打开浏览器访问 `http://localhost:3000/admin`
2. 输入用户名 `admin`，密码 `admin123`
3. 点击「登录」按钮
4. 登录成功后进入数据概览页面

### 步骤2：查看待处理订单

1. 点击左侧菜单「订单管理」
2. 在筛选栏选择状态「待处理」
3. 查看待审核的订单列表
4. 关注「待处理订单」数量（首页统计卡片也会显示）

### 步骤3：核验卡券信息

1. 在订单列表中点击「详情」按钮
2. 查看订单详情弹窗中的**卡券信息**区域
3. 核对卡号（`cardNoDecrypt`）和卡密（`cardPwdDecrypt`）是否有效
4. 查看用户信息，确认用户身份

### 步骤4：完结或拒绝

**确认完结：**
1. 核验无误后，点击「确认完结」按钮
2. 系统弹出确认提示
3. 确认后订单状态变为「已完成」，金额自动入账用户余额
4. 首页统计数据自动更新

**拒绝订单：**
1. 核验发现问题时，点击「拒绝订单」按钮
2. 在弹窗中填写拒绝原因（如：卡密错误、卡号不存在等）
3. 确认后订单状态变为「已失败」
4. 小程序端用户会看到失败原因

### 步骤5：查看用户信息

1. 点击左侧菜单「用户管理」
2. 搜索或浏览用户列表
3. 点击「详情」查看用户余额、订单统计、余额流水

## 九、安全说明

1. **管理员密码**使用 bcrypt 加密存储，不可逆
2. **管理员Token**使用 JWT，7天过期，每次请求需携带
3. **卡号卡密**在数据库中使用 AES-256-CBC 加密存储，管理后台查看时解密显示
4. **完结订单**使用数据库事务，保证余额入账和流水记录的一致性
5. **管理后台API**所有接口（除登录外）均需管理员Token认证
6. **管理后台前端**内置在后端服务中，通过 `/admin` 路径访问，无需单独部署

## 十、常见问题

### Q1：数据库连接失败？

检查 `server/.env` 中的数据库配置是否正确，确认 MySQL 服务已启动，确认密码正确。

### Q2：管理员密码忘了？

重新执行 `init.sql` 中的管理员 INSERT 语句，或手动更新数据库中 admin 表的 password 字段。

生成新密码的 bcrypt 哈希：
```javascript
const bcrypt = require('bcryptjs');
const hash = bcrypt.hashSync('新密码', 10);
console.log(hash);
```

### Q3：管理后台页面打不开？

1. 确认后端服务已启动（`npm run dev`）
2. 确认访问地址为 `http://localhost:3000/admin`
3. 检查浏览器控制台是否有错误

### Q4：完结订单时报错「订单状态异常」？

只有状态为 `PENDING`（待处理）或 `PROCESSING`（处理中）的订单可以完结。已完成、已失败、已取消的订单无法再次操作。

### Q5：小程序端订单状态没有更新？

小程序端目前使用本地存储（localStorage）管理订单数据。当后端服务正式上线后，小程序会通过 API 实时拉取后端订单状态，状态变更后自动显示。在 LOCAL_DEV 模式下，订单状态由本地模拟数据决定。

### Q6：如何新增管理员？

目前需通过数据库直接操作：
```sql
INSERT INTO admin (username, password, real_name, role) 
VALUES ('新用户名', 'bcrypt加密的密码', '真实姓名', 'OPERATOR');
```

后续可开发管理员管理页面（SUPER_ADMIN 权限）。
