# 意见反馈与关于我们功能设计文档

## 一、功能概述

### 1.1 问题分析

当前"意见反馈"和"关于我们"功能均跳转到 webview 页面，但 webview 依赖的外部网页 URL 为占位地址（`https://www.example.com/feedback`、`https://www.example.com/about`），实际无法加载，导致功能不可用。

### 1.2 解决方案

将两个功能从 webview 外部网页改为**原生小程序页面**实现，彻底摆脱对外部网页的依赖，同时提供更丰富的交互体验。

### 1.3 功能目标

**意见反馈**：
- 支持用户选择反馈类型（功能建议、体验问题、Bug反馈、其他）
- 支持文字描述 + 图片上传（最多3张）
- 支持联系方式填写
- 提交后展示历史反馈记录及处理状态

**关于我们**：
- 展示公司/产品介绍信息
- 展示应用版本信息
- 提供用户协议、隐私政策等法律文档入口
- 提供联系方式快捷入口

### 2.2 交互流程

```
用户点击"意见反馈"
    │
    ▼
进入意见反馈页面
    │
    ├─ 选择反馈类型（必选，默认"功能建议"）
    ├─ 输入反馈内容（必填，≥10字）
    ├─ 上传图片（可选，0-3张）
    ├─ 输入联系方式（可选）
    │
    ▼
点击"提交反馈"
    │
    ├─ 校验失败 → Toast提示具体错误
    │
    ▼
校验通过，提交到后端
    │
    ├─ 成功 → Toast"反馈已提交"，清空表单，刷新反馈记录
    │
    └─ 失败 → Toast"提交失败，请稍后重试"
```

## 四、接口设计

### 4.1 提交意见反馈

**接口路径**：`POST /common/feedback`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 反馈类型：suggestion/experience/bug/other |
| content | string | 是 | 反馈内容（≥10字） |
| images | string[] | 否 | 图片URL列表（0-3张） |
| contact | string | 否 | 联系方式 |
| userId | string | 否 | 用户ID |

**响应数据**：

```json
{
  "code": 200,
  "data": {
    "id": "FB202401010001",
    "status": "pending"
  }
}
```

### 4.2 获取反馈记录列表

**接口路径**：`GET /common/feedback/list`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | string | 是 | 用户ID |
| page | number | 否 | 页码，默认1 |
| pageSize | number | 否 | 每页数量，默认10 |

**响应数据**：

```json
{
  "code": 200,
  "data": {
    "list": [...],
    "total": 5,
    "page": 1,
    "pageSize": 10
  }
}
```

## 五、与现有模块的集成

### 5.1 个人中心页面修改

```javascript

  wx.navigateTo({
    url: '/pages/webview/webview?type=feedback&title=意见反馈'
  });
}
onAbout() {
  wx.navigateTo({
    url: '/pages/webview/webview?type=about&title=关于我们'
  });

// 修改后 - 跳转原生页面
onFeedback() {
  wx.navigateTo({
    url: '/pages/feedback/feedback'
  });
}

  wx.navigateTo({
    url: '/pages/about/about'
  });
}
```
