# 小程序-手机号登录需求文档

## 1. 背景与问题

### 1.1 当前现状

当前小程序手机号登录采用"手动输入手机号 + 密码/验证码"的方式，存在以下问题：

| 问题 | 说明 |
|------|------|
| **体验差** | 用户需手动输入11位手机号，易输错，操作繁琐 |
| **不安全** | 手动输入手机号+验证码的方式，无法确认手机号真实归属，存在冒用风险 |
| **不符合微信规范** | 微信官方推荐使用「手机号快速验证」组件，一键获取微信绑定的手机号 |
| **逻辑混乱** | 密码登录和验证码登录并存，新用户/老用户判断逻辑复杂，密码设置流程嵌入登录流程不合理 |
| **短信成本高** | 自建短信验证码通道成本高，且存在到达率问题 |

### 1.2 微信官方推荐方式

微信小程序提供两种手机号获取组件：

| 组件 | 能力 | 费用 | 适用场景 |
|------|------|------|---------|
| `<button open-type="getPhoneNumber">` | 获取微信绑定的手机号（需用户主动点击授权） | 每次调用消耗1个手机号快速验证额度 | **登录/注册**（推荐） |
| `<button open-type="getRealtimePhoneNumber">` | 获取用户当前使用的手机号（实时验证） | 每次调用消耗1个手机号实时验证额度 | 高安全场景（金融、支付） |

> **本方案采用 `getPhoneNumber`**，满足登录场景需求，成本更低。

## 2. 需求概述

### 2.1 核心目标

按照微信小程序手机号登录的标准方式，重新设计手机号登录功能：

1. **一键获取手机号**：用户点击按钮 → 微信弹出授权 → 自动获取手机号 → 完成登录/注册
2. **无密码设计**：手机号登录不设密码，后续可通过微信手机号组件再次验证身份
3. **新老用户无感**：新用户自动注册，老用户直接登录，流程完全一致
4. **保留微信登录**：微信一键登录（wx.login）作为备选登录方式

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **极简** | 用户只需1次点击即可完成手机号登录 |
| **安全** | 手机号由微信服务端验证，无法伪造 |
| **无密码** | 取消手机号登录的密码体系，降低用户记忆负担 |
| **兼容** | 已有密码登录的老用户不受影响，密码登录作为独立入口保留 |

### 3.2 手机号快捷登录流程

#### 3.2.1 主流程（正常情况）

```
用户点击「微信手机号快捷登录」按钮
        │
        ▼
微信弹出授权弹窗（"XX申请获取你的手机号"）
        │
    ┌───┴───┐
    │ 允许   │ 拒绝
    ▼       ▼
 获取code  提示"需要手机号才能登录"
    │
    ▼
前端将 code 发送到后端
    │
    ▼
后端调用微信API解密手机号
    │
    ▼
查询数据库：该手机号是否已注册？
    │
  ┌─┴──┐
  │是   │否
  ▼    ▼
直接登录  自动注册后登录
  │    │
  └─┬──┘
    ▼
返回 token + userInfo
    │
    ▼
前端存储登录信息，跳转首页
```

#### 3.2.2 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 用户拒绝授权 | 提示"需要手机号才能登录"，停留在登录页 |
| 微信API调用失败 | 提示"获取手机号失败，请重试" |
| code过期（5分钟） | 提示"授权已过期，请重新点击" |
| 网络异常 | 提示"网络异常，请重试" |
| 后端解密失败 | 提示"获取手机号失败，请重试" |

### 3.3 新老用户处理

| 用户类型 | 判断方式 | 处理逻辑 |
|---------|---------|---------|
| **新用户**（手机号未注册） | `User.findOne({ where: { phone } })` 返回 null | 自动创建用户，nickname 设为"用户+手机尾号4位"，loginType 设为 `PHONE_WX`，**不设置密码** |
| **老用户**（手机号已注册） | 查询到用户记录 | 直接登录，更新 openId 关联（如之前是微信登录用户，补充手机号） |

### 3.4 与微信登录的关系

| 场景 | 处理方式 |
|------|---------|
| 用户先用微信登录（无手机号），后用手机号快捷登录 | 通过 openId 匹配，补充手机号到已有用户记录 |
| 用户先用手机号快捷登录，后用微信登录 | 通过 openId 匹配，补充 openId 到已有用户记录 |
| 同一手机号 + 不同 openId | 以手机号为主键，更新 openId |

#### 4.1.2 核心JS逻辑

```javascript
/**
 * 微信手机号快捷登录
 */
async onGetPhoneNumber(e) {
  // 用户拒绝授权
  if (e.detail.errMsg !== 'getPhoneNumber:ok') {
    wx.showToast({ title: '需要手机号才能登录', icon: 'none' });
    return;
  }

  // 检查协议
  if (!this.data.agreed) {
    wx.showToast({ title: '请先同意用户协议', icon: 'none' });
    return;
  }

  try {
    this.setData({ loading: true });

    // 1. 先调用 wx.login 获取 code
    const loginRes = await wx.login();

    // 2. 将 loginCode + phoneCode 发送到后端
    const data = await request.post(API.user.phoneQuickLogin, {
      code: loginRes.code,           // wx.login 的 code（用于获取 openId）
      phoneCode: e.detail.code       // getPhoneNumber 的 code（用于解密手机号）
    });

    // 3. 存储登录信息
    setLoginInfo(data.token, data.userInfo);

    wx.showToast({ title: '登录成功', icon: 'success' });

    setTimeout(() => {
      this.handleLoginSuccess(data.userInfo);
    }, 1500);
  } catch (err) {
    wx.showToast({ title: err.message || '登录失败', icon: 'none' });
  } finally {
    this.setData({ loading: false });
  }
}
```

### 4.2 后端接口设计

#### 4.2.1 新增接口：手机号快捷登录

| 项目 | 说明 |
|------|------|
| **URL** | `POST /api/user/phoneQuickLogin` |
| **描述** | 通过微信手机号组件获取的手机号进行快捷登录/注册 |
| **认证** | 无需（公开接口） |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | string | 是 | wx.login() 返回的 code，用于获取 openId |
| phoneCode | string | 是 | getPhoneNumber 返回的 code，用于解密手机号 |

**响应示例：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "userInfo": {
      "id": 1,
      "nickname": "用户1234",
      "avatar": "",
      "phone": "138****1234",
      "isVerified": false
    }
  }
}
```

**后端处理流程：**

```
1. 用 code 调用微信 jscode2session API → 获取 openId + session_key
2. 用 phoneCode + session_key 调用微信 getPhoneNumber API → 获取手机号
3. 查询数据库：该手机号是否已注册？
   a. 已注册 → 更新 openId（如不同），直接返回 token
   b. 未注册 → 创建新用户，返回 token
4. 返回 token + userInfo
```

**后端核心代码逻辑：**

```javascript
exports.phoneQuickLogin = async (req, res, next) => {
  try {
    const { code, phoneCode } = req.body;

    // 1. 获取微信 openId 和 session_key
    const wxData = await userService.getWxSession(code);

    // 2. 解密手机号
    const phoneResult = await userService.getPhoneNumber(phoneCode);

    // 3. 查找或创建用户
    let user = await userService.findByPhone(phoneResult.phoneNumber);
    const isNewUser = !user;

    if (!user) {
      // 新用户自动注册
      user = await userService.create({
        openId: wxData.openid,
        phone: phoneResult.phoneNumber,
        nickname: '用户' + phoneResult.phoneNumber.slice(-4),
        loginType: 'PHONE_WX'
      });
    } else {
      // 老用户：更新 openId 关联
      if (!user.openId || user.openId !== wxData.openid) {
        await userService.update(user.id, { openId: wxData.openid });
      }
    }

    // 4. 生成 token
    const token = generateToken({ userId: user.id });

    res.json({
      code: 200,
      message: isNewUser ? '注册并登录成功' : '登录成功',
      data: {
        token,
        userInfo: {
          id: user.id,
          nickname: user.nickname,
          avatar: user.avatar,
          phone: user.phone ? user.phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2') : '',
          isVerified: user.isVerified
        }
      }
    });
  } catch (err) {
    logger.error('手机号快捷登录失败:', err);
    next(err);
  }
};
```

#### 4.2.2 新增 Service 方法：getPhoneNumber

微信从基础库 2.21.2 开始，`getPhoneNumber` 返回的 `code` 需通过后端调用微信 API 换取手机号：

| 项目 | 说明 |
|------|------|
| **微信API** | `POST https://api.weixin.qq.com/wxa/business/getuserphonenumber?access_token=ACCESS_TOKEN` |
| **请求参数** | `code`（前端 getPhoneNumber 返回的 code） |
| **返回** | `{ "phone_info": { "phoneNumber": "13800138000", "purePhoneNumber": "13800138000", "countryCode": "86" } }` |

```javascript
// server/src/services/user.js 新增方法
exports.getPhoneNumber = async (phoneCode) => {
  // 1. 获取 access_token
  const accessToken = await getAccessToken();

  // 2. 调用微信接口获取手机号
  const url = `https://api.weixin.qq.com/wxa/business/getuserphonenumber?access_token=${accessToken}`;
  const response = await axios.post(url, { code: phoneCode });

  if (response.data.errcode !== 0) {
    throw new Error(response.data.errmsg || '获取手机号失败');
  }

  return response.data.phone_info;
};
```

#### 4.2.3 access_token 管理

| 项目 | 说明 |
|------|------|
| **获取方式** | `GET https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=APPID&secret=APPSECRET` |
| **有效期** | 2小时 |
| **缓存策略** | 服务端缓存，过期前10分钟自动刷新 |
| **存储方式** | 内存变量（单实例）/ Redis（多实例） |

```javascript
// server/src/utils/wxToken.js
let tokenCache = { token: '', expireTime: 0 };

exports.getAccessToken = async () => {
  // 未过期直接返回
  if (tokenCache.token && Date.now() < tokenCache.expireTime) {
    return tokenCache.token;
  }

  // 重新获取
  const url = `https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=${process.env.WX_APPID}&secret=${process.env.WX_SECRET}`;
  const res = await axios.get(url);

  if (res.data.errcode) {
    throw new Error('获取access_token失败: ' + res.data.errmsg);
  }

  tokenCache = {
    token: res.data.access_token,
    expireTime: Date.now() + (res.data.expires_in - 600) * 1000  // 提前10分钟过期
  };

  return tokenCache.token;
};
```

## 5. 移除/保留的功能

### 5.1 移除的功能

| 功能 | 原因 |
|------|------|
| 手动输入手机号 | 由微信手机号组件替代，无需手动输入 |
| 验证码登录（短信验证码） | 由微信手机号组件替代，无需短信通道 |
| 密码登录 | 手机号登录不再需要密码，简化流程 |
| 新用户设置密码 | 无密码设计，取消密码体系 |
| 发送验证码按钮 | 不再需要 |
| 验证码倒计时 | 不再需要 |
| 手机号登录子Tab切换（密码/验证码） | 不再需要 |

### 5.2 保留的功能

| 功能 | 说明 |
|------|------|
| 微信一键登录（wx.login） | 作为备选登录方式保留 |
| 开发环境登录（devLogin） | 开发调试用，保留 |
| 协议勾选 | 保留，未勾选时禁用登录按钮 |
| 游客模式 | 保留"暂不登录，先逛逛" |
| 实名认证 | 不受影响，登录后仍需实名认证 |

### 5.3 数据库影响

| 表 | 字段 | 影响 |

## 6. 安全设计

### 6.1 安全保障

| 安全点 | 说明 |
|--------|------|
| 手机号真实性 | 由微信服务端验证，无法伪造 |
| 传输安全 | code 有效期仅5分钟，且一次性使用 |
| 防重放攻击 | code 使用后即失效 |
| 隐私保护 | 前端展示手机号时脱敏（138****1234） |
| 协议确认 | 用户必须勾选协议才能点击登录按钮 |

### 6.2 与旧方案安全对比

| 对比项 | 旧方案（手动输入+验证码） | 新方案（微信手机号组件） |
|--------|------------------------|----------------------|
| 手机号真实性 | 依赖短信验证，存在短信拦截风险 | 微信服务端验证，无法伪造 |
| 验证码安全 | 可能被拦截/转发 | 无验证码，无此风险 |
| 短信轰炸 | 需做频率限制 | 不涉及 |
| 密码泄露 | MD5加密，安全性较低 | 无密码，无此风险 |

async onGetPhoneNumber(e) {
