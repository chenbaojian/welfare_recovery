# 后端管理平台 - 已销售资产和可销售资产需求文档

## 1. 功能概述

在管理后台侧边栏「回收资产」下方新增两个子栏目：

- **已销售资产** — 统计和展示所有已被买家购买的卡券资产数据（`order` 表中 `status='SUCCESS' AND is_sold=1`），帮助管理员掌握卡券销售情况、买家信息和销售收入。
- **可销售资产** — 统计和展示所有已完成回收且尚未售出的卡券资产数据（`order` 表中 `status='SUCCESS' AND is_sold=0`），帮助管理员掌握当前可售库存、面值分布和潜在销售收入。

两个页面参考现有「回收资产」栏目的设计风格和交互模式，保持管理后台整体一致性。

---

## 2. 用户角色

| 角色 | 权限 |
|------|------|
| 超级管理员（SUPER_ADMIN） | 可查看、导出已销售/可销售资产数据 |
| 管理员（ADMIN） | 可查看、导出已销售/可销售资产数据 |
| 运营人员（OPERATOR） | 仅可查看已销售/可销售资产数据，不可导出 |

---

## 3. 页面结构

### 3.1 导航位置

在管理后台侧边栏菜单中，在「回收资产」下方新增两个菜单项：

```
📊 数据概览
📋 订单管理
👥 用户管理
💰 回收资产
🛒 已销售资产    ← 新增
📦 可销售资产    ← 新增
```

菜单图标与文本：
- 已销售资产：图标🛒，菜单文本：已销售资产，`data-page="soldAssets"`，`data-menu="soldAssets"`
- 可销售资产：图标📦，菜单文本：可销售资产，`data-page="availableAssets"`，`data-menu="availableAssets"`

### 3.2 页面布局

两个页面均采用与「回收资产」相同的布局结构：

1. **顶部统计概览区** — 汇总卡片，展示全局统计
2. **筛选栏** — 支持多维度筛选
3. **卡券分类汇总表格** — 按卡券类型分组展示
4. **面值分布区域** — 展开查看某类型的面值分布
5. **卡券明细弹窗** — 点击查看明细，展示完整卡券信息

## 4. 已销售资产 - 功能详细说明

### 4.1 统计概览区

页面顶部展示 4 个统计卡片，数据来源为所有 `status='SUCCESS' AND is_sold=1` 的订单：

| 卡片 | 数据字段 | 说明 |
|------|----------|------|
| 已售总张数 | `totalCount` | 已售出卡券的总数量 |
| 已售总面值 | `totalFaceValue` | 所有已售出卡券的 `faceValue` 之和 |
| 销售总收入 | `totalBuyAmount` | 所有已售出卡券对应的 `buy_order.buy_price` 之和（买家实际支付金额） |
| 卡券种类数 | `cardTypeCount` | 涉及的不同 `cardTypeName` 种类数量 |

**示例展示：**

```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  🛒 已售总张数 │ │  💵 已售总面值 │ │  💰 销售总收入 │ │  🎫 卡券种类数 │
│      68      │ │  ¥6,800.00  │ │  ¥6,120.00  │ │      5      │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

### 4.2 卡券分类汇总表格

按卡券类型（`cardTypeName`）分组展示：

| 字段 | 说明 |
|------|------|
| 卡券类型名称 | `cardTypeName` |
| 已售张数 | 该类型下已售出卡券的数量 |
| 总面值金额 | 该类型下所有已售出卡券 `faceValue` 的合计 |
| 总回收金额 | 该类型下所有已售出卡券 `recycleAmount` 的合计（卖家回收所得） |
| 总销售金额 | 该类型下所有已售出卡券对应的 `buy_price` 合计（买家支付金额） |
| 平均折扣率 | `总回收金额 / 总面值金额 × 100%`，反映该类卡券的平均回收折扣 |
| 操作 | 「面值分布」「查看明细」按钮 |

### 4.3 面值分布统计

与回收资产页面一致，按面值维度展示分布：

| 面值 | 已售张数 | 总面值 | 总回收金额 | 总销售金额 |
|------|----------|--------|-----------|-----------|

### 4.4 卡券明细弹窗

点击「查看明细」弹出明细弹窗，展示已售出卡券的完整信息：

| 列名 | 数据来源 | 说明 |
|------|----------|------|
| 回收订单号 | `order.orderNo` | 卖家提交的回收订单号 |
| 卖家昵称 | 关联 User.nickname | 提交回收订单的卖家昵称 |
| 卖家手机号 | 关联 User.phone | 提交回收订单的卖家手机号 |
| 卡券面值 | `order.faceValue` | 卡券原始面值 |
| 回收金额 | `order.recycleAmount` | 卖家实际回收所得 |
| 购买价格 | `buy_order.buyPrice` | 买家实际支付金额 |
| 买家昵称 | 关联 BuyOrder.buyerId → User.nickname | 购买该卡券的买家昵称 |
| 买家手机号 | 关联 BuyOrder.buyerId → User.phone | 购买该卡券的买家手机号 |
| 购买订单号 | `buy_order.orderNo` | 买家的购买订单号 |
| 卡号 | `order.cardNo`（解密后） | 卡券卡号 |
| 卡密 | `order.cardPwd`（解密后） | 卡券卡密 |
| 完结时间 | `order.completeTime` | 管理员确认完结的时间 |
| 售出时间 | `buy_order.payTime` 或 `buy_order.createTime` | 买家购买/支付的时间 |

**安全要求：** 卡号和卡密属于敏感信息，页面需标注「⚠️ 卡号、卡密为敏感信息，仅管理员可见，请勿外泄」提示。

### 4.5 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 卡券类型 | 下拉选择 | 筛选特定卡券类型，选项从 `card_type` 表动态获取 |
| 面值范围 | 数值输入 | 最小面值 ~ 最大面值 |
| 时间范围 | 日期范围选择器 | 按售出时间（`buy_order.createTime`）筛选 |
| 卖家手机号 | 文本输入 | 搜索卖家的手机号 |
| 买家手机号 | 文本输入 | 搜索买家的手机号 |

## 7. 数据查询逻辑

### 7.1 已销售资产核心查询条件

```sql
WHERE o.status = 'SUCCESS' AND o.is_sold = 1
```

已销售资产需要关联 `buy_order` 表获取买家信息和购买价格：

```sql
SELECT o.*, bo.buy_price, bo.order_no AS buy_order_no, bo.buyer_id,
       bo.create_time AS sold_time, bo.pay_time
FROM `order` o
LEFT JOIN buy_order bo ON o.buy_order_id = bo.id
WHERE o.status = 'SUCCESS' AND o.is_sold = 1
  [AND 筛选条件]
```

### 7.2 可销售资产核心查询条件

```sql
WHERE status = 'SUCCESS' AND is_sold = 0
```

可销售资产需要关联 `card_type` 表获取折扣率以计算预估销售金额：

```sql
SELECT o.*, ct.discount_rate
FROM `order` o
LEFT JOIN card_type ct ON o.card_type_id = ct.id
WHERE o.status = 'SUCCESS' AND o.is_sold = 0
  [AND 筛选条件]
```

预估销售金额计算：`faceValue × discountRate`

### 7.3 已销售资产分类汇总查询

```sql
SELECT
  o.card_type_name,
  COUNT(*) AS total_count,
  SUM(o.face_value) AS total_face_value,
  SUM(o.recycle_amount) AS total_recycle_amount,
  SUM(bo.buy_price) AS total_buy_amount,
  ROUND(SUM(o.recycle_amount) / SUM(o.face_value) * 100, 2) AS avg_discount_rate
FROM `order` o
LEFT JOIN buy_order bo ON o.buy_order_id = bo.id
WHERE o.status = 'SUCCESS' AND o.is_sold = 1
  [AND 筛选条件]
GROUP BY o.card_type_name
ORDER BY total_count DESC
```

### 7.4 可销售资产分类汇总查询

```sql
SELECT
  o.card_type_name,
  COUNT(*) AS total_count,
  SUM(o.face_value) AS total_face_value,
  SUM(o.recycle_amount) AS total_recycle_amount,
  SUM(o.face_value * ct.discount_rate) AS estimated_buy_amount,
  ROUND(SUM(o.recycle_amount) / SUM(o.face_value) * 100, 2) AS avg_discount_rate
FROM `order` o
LEFT JOIN card_type ct ON o.card_type_id = ct.id
WHERE o.status = 'SUCCESS' AND o.is_sold = 0
  [AND 筛选条件]
GROUP BY o.card_type_name
ORDER BY total_count DESC
```

### 7.5 买家手机号筛选

已销售资产支持按买家手机号筛选，需通过 `buy_order.buyer_id` 关联 `user` 表：

```sql
-- 先查出匹配手机号的买家ID列表
SELECT id FROM user WHERE phone LIKE '%手机号%';
-- 再筛选 buy_order.buyer_id IN (买家ID列表)
-- 最终筛选 o.id IN (SELECT recycle_order_id FROM buy_order WHERE buyer_id IN (买家ID列表))
```

## 11. 性能考虑

| 场景 | 优化方案 |
|------|----------|
| 大量订单数据聚合 | 使用数据库 `GROUP BY` + `SUM` 聚合查询 |
| 明细列表分页 | 使用 `LIMIT` + `OFFSET` 分页查询，默认每页20条 |
| 用户信息关联 | 批量查询用户信息，避免N+1查询 |
| 关联 buy_order/card_type | 批量查询关联表数据，内存中映射 |
| 导出大数据量 | 超过1000条时采用流式写入Excel |
| 索引利用 | 利用 `idx_status_sold` 复合索引高效查询 |

```
  │     │     ├── nickname → 买家昵称
  │     │     └── phone    → 买家手机号
  │     ├── createTime → 购买时间
  │     └── payTime    → 支付时间
  └── orderNo         → 回收订单号
```

### 14.2 可销售资产

```
Order (status='SUCCESS' AND is_sold=0)
  ├── cardTypeName    → 卡券类型名称（分组维度）
  ├── faceValue       → 卡券面值
  ├── recycleAmount   → 回收金额（卖家所得）
  ├── cardNo/cardPwd  → 卡号卡密（加密，展示时解密）
  ├── completeTime    → 完结时间
  ├── userId          → 关联卖家 User
  │     ├── nickname  → 卖家昵称
  │     └── phone     → 卖家手机号
  ├── cardTypeId      → 关联 CardType
  │     ├── discountRate → 购买折扣率
  │     └── name         → 卡券类型名称
  └── orderNo         → 回收订单号

预估销售金额 = faceValue × CardType.discountRate
```
